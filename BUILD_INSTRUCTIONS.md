# Инструкции по сборке проекта Neon Arena

## Требования

- **Embarcadero RAD Studio** (C++ Builder) версии 10.3 или выше
- Windows 7/8/10/11

## Пошаговая инструкция

### 1. Открытие проекта

1. Запустите RAD Studio
2. Выберите `File > Open Project...`
3. Найдите и откройте файл `Project1.cbproj`

### 2. Настройка путей (если необходимо)

Если проект не компилируется из-за ошибок путей:

1. В RAD Studio откройте `Project > Options...`
2. Перейдите в раздел `C++ Compiler > Paths and Defines`
3. Проверьте пути в `Include Path` - они должны указывать на папку с исходными файлами
4. Если нужно, добавьте путь к папке проекта вручную

### 3. Выбор конфигурации

1. В верхней панели инструментов выберите:
   - **Конфигурация:** Debug (для отладки) или Release (для финальной версии)
   - **Платформа:** Win32 (32-bit) или Win64 (64-bit)

### 4. Компиляция

**Вариант A: Компиляция и запуск**
- Нажмите `F9` или кнопку "Run" (зелёная стрелка)
- Проект будет скомпилирован и запущен автоматически

**Вариант B: Только компиляция**
- Выберите `Project > Build All` или нажмите `Ctrl+F9`
- Исполняемый файл будет создан в папке `Win32\Debug\` (или `Win32\Release\`)

### 5. Результат

После успешной компиляции:
- Исполняемый файл: `Win32\Debug\Project1.exe` (или `Win32\Release\Project1.exe`)
- Промежуточные файлы: `.obj`, `.tds`, `.pch` и другие в той же папке

## Компиляция из командной строки

Если у вас установлен RAD Studio, вы можете компилировать проект из командной строки:

```batch
msbuild Project1.cbproj /p:Config=Debug /p:Platform=Win32
```

Или используя встроенный компилятор:

```batch
bcc32 -c Project1.cpp
```

## Решение проблем

### Ошибка: "rtl290.bpl не найден" при запуске
**Важно:** Проект настроен на статическую линковку RTL, но если вы видите эту ошибку:

1. Откройте `Project > Options...`
2. Перейдите в `C++ Compiler > Classic Compiler` (или `C++ Linker`)
3. Убедитесь, что **"Dynamic RTL"** выключена (false)
4. Убедитесь, что **"Use runtime packages"** выключена (false)
5. Перекомпилируйте проект

**Проверка в файле проекта:**
- Откройте `Project1.cbproj` в текстовом редакторе
- Найдите `<DynamicRTL>` - должно быть `false`
- Найдите `<UsePackages>` - должно быть `false`

### Ошибка: "Cannot find include file"
- Проверьте, что все `.h` файлы находятся в папке проекта
- Убедитесь, что пути в настройках проекта корректны

### Ошибка: "Unresolved external"
- Убедитесь, что все `.cpp` файлы добавлены в проект
- Проверьте, что все необходимые библиотеки подключены

### Ошибка: "Linker error"
- Проверьте настройки линковщика в `Project > Options > C++ Linker`
- Убедитесь, что все необходимые библиотеки указаны

### Проект компилируется, но не запускается
- Проверьте, что исполняемый файл создан
- Попробуйте запустить `.exe` напрямую из папки `Win32\Debug\`
- Проверьте логи ошибок в RAD Studio
- Убедитесь, что используется статическая RTL (см. выше)

## Создание Release версии

Для создания оптимизированной версии:

1. Выберите конфигурацию **Release**
2. Выберите `Project > Build All`
3. Исполняемый файл будет в `Win32\Release\Project1.exe`

Release версия:
- Оптимизирована по скорости
- Не содержит отладочной информации
- Имеет меньший размер файла
